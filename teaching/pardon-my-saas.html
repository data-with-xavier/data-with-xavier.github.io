Before we write a single line of code, let's talk about why you're here — and why the tools your company probably uses are working against you.

The SaaS Tax

Tableau Desktop runs $840/year per user. Power BI Pro is $120/year but requires a Microsoft 365 ecosystem. Looker? Enterprise pricing — call for a quote (which means very expensive). Every year, companies pay these "SaaS taxes" for tools that lock your work into proprietary formats, limit your design choices, and slow you down the moment you want to do something outside their feature set.

The lock-in is the real cost. Your Tableau workbooks can't run without Tableau. Your Power BI dashboards live in Microsoft's cloud. You're renting access to your own work.

The AI Alternative

HTML is the language of the web. It's free, runs everywhere, and looks exactly how you design it. The catch has always been: you need to know how to write it. That's where Claude changes everything.

You describe what you want. Claude writes the code. You refine it by describing what to change. No syntax memorization. No Stack Overflow spirals. Just a conversation that produces something you can publish.

This isn't "AI does everything while you watch." You are the analyst, the designer, the storyteller. Claude is the developer who executes your vision faster than any team could.

What "Pardon My SaaS" Means

It means you're done apologizing for not knowing Tableau. It means you're building something better, on your own terms, that you actually own. By the end of this course, you'll have a portfolio of live, published dashboards — not screenshots from someone else's platform.

Getting Unstuck — The Most Important Skill in This Course

Claude will sometimes give you broken code. This is normal. Here's the loop that fixes it every time:

  1. Open the HTML file in your browser — right-click on the file, choose Open with → your browser
  2. Right-click anywhere on the page → Inspect → click the Console tab
  3. Copy any red error messages you see
  4. Tell Claude exactly what's wrong — paste the error and describe what you see
Prompt — When Something Breaks
This code isn't working as expected. Here's what I see:

[Describe what you see visually — e.g., "the page is blank" or "the chart doesn't appear"]

Here's the error from the browser console:
[Paste the error message]

Please identify the problem and fix it.
The rule: Be specific. "Fix it" gives Claude nothing. "The bar chart labels are overlapping and the font size is too small — here's the console error: [error]" gives Claude everything it needs. The more specific your description, the better the fix.

You don't need to learn HTML, CSS, or JavaScript to build great dashboards. But you do need to understand what they are — enough to describe what you want clearly and recognize when something is off.

What Is an HTML File?

An HTML file is a single text file that any browser can render as a webpage. When we build dashboards, everything lives in one file: the structure (HTML), the styling (CSS), and the interactivity (JavaScript). You can open it locally by double-clicking it, or publish it to GitHub Pages and share the URL with anyone.

You don't write this file. Claude does. Your job is to describe what you want clearly enough that Claude can build it — and then to refine it until it's right.

What Is Chart.js?

Chart.js is a free, open-source JavaScript library that creates beautiful, interactive charts. We use it by loading it from a CDN (content delivery network) — one line of code at the top of our HTML file, no installation required.

Claude knows Chart.js well. When you ask for a bar chart, line chart, doughnut, scatter plot, or radar chart, Claude will use Chart.js to build it.

Working with Data — Two Approaches

Approach 1: Embed data in the HTML — Best for small, static datasets. Claude writes the data directly into the JavaScript as a variable. Simple, portable, works everywhere.

Approach 2: Fetch from GitHub — Best for larger datasets or data you want to update easily. You upload the CSV to your GitHub repo, then the HTML fetches it via the raw URL. This is what we'll do in the projects.

Why not just open a CSV locally? Browsers block local file access for security reasons. By hosting your data on GitHub and fetching it via URL, you bypass this entirely — and get version control for free.

After Project 1, you've felt what it's like to start from scratch. Every new Claude conversation starts cold — it doesn't know your color palette, your layout preferences, or your visual style. Your dashboard spec fixes that permanently.

What Is a Dashboard Spec?

A dashboard-spec.md file that lives in your GitHub repo. It defines your personal design system: colors, fonts, layout preferences, chart style, and the overall mood of your work. You paste it at the start of every Claude session, and Claude uses it as the foundation for everything it builds.

The result: your dashboards will look like yours — consistent, branded, intentional — instead of whatever style Claude defaults to.

Download dashboard-spec-template.md

Please ask me a series of questions to understand my design preferences — things like:
- Colors or palettes I'm drawn to (ask for examples if helpful)
- Layouts I find appealing (minimal sidebar? full-screen? card-heavy?)
- The mood or personality I want my work to convey (corporate and clean? bold and data-forward? editorial and calm?)

  • Cursor — An AI-powered code editor that's excellent for larger projects with multiple files. Great next step if you want to build more complex apps.
